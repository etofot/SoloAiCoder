# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: ИНТЕРАКТИВНАЯ МАТРИЦА СВЯЗЕЙ (CYBERPUNK SKILL MATRIX)

## 1. ОБЩЕЕ ОПИСАНИЕ
Необходимо разработать и интегрировать высокотехнологичную систему интерактивных связей для веб-резюме. Система должна визуализировать зависимости между навыками (Skills) и опытом работы (Experience) с использованием эстетики микроэлектроники и футуристичных интерфейсов (стиль Cyberpunk / High-Tech).

**Цель:** Создать «WOW-эффект» при просмотре страницы, имитируя работу нейронной сети или лазерную гравировку дорожек на микропроцессоре.

---

## 2. ВИЗУАЛЬНЫЙ СТИЛЬ
- **Геометрия:** Только ломаные линии с прямыми углами (Orthogonal Routing). Никаких плавных кривых.
- **Эстетика:** Дорожки на печатной плате, лазерные лучи, выжигание кремния.
- **Анимация:** 
    - Появление линии: «прорастание» из точки А в точку Б.
    - «Фотон»: яркий световой сгусток (голова линии), который движется по траектории, оставляя за собой пульсирующий след.
    - Состояние покоя: после отрисовки линия должна едва заметно пульсировать (пунктир «муравьиная дорожка»).
- **Цвет:** Электрический циановый (`#00f5ff`) и неоновый синий (`#4d5dfb`).

---

## 3. КЛЮЧЕВЫЕ ТРЕБОВАНИЯ К ЛОГИКЕ
### 3.1. Умная трассировка (Smart Routing)
- Линии **не должны накладываться** друг на друга. Если от одной ноды идет несколько связей, они должны идти параллельно с небольшим смещением (как шлейф проводов).
- Пути должны **огибать** препятствия (кнопки, заголовки).
- Использовать «внешние желоба» (gutters) для обхода основного контента.

### 3.2. Расширение области действия
- Сейчас связи работают только между кнопками в блоке Skills. 
- **Задача:** Реализовать связи между текстовыми блоками в разделе «Опыт работы» и навыками.
    - *Пример:* При наведении на текст «Разработка аудиодвижков и DSP» должна простроиться линия к кнопке «Node Systems» или «Automation».
    - *Пример:* «Проектирую 3D-визуальную платформу» должна соединяться с «Dialogue Auto-responder».

### 3.3. Триггеры (Triggers)
- **Hover:** Линии появляются при наведении на ключевые слова или блоки.
- **Scroll Tracking:** При пролистывании страницы (чтении конкретного раздела опыта) связи этого раздела должны активироваться автоматически (рекомендуется `IntersectionObserver`).

---

## 4. ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ И СТЕК
- **Среда:** Статический HTML/JS/CSS (GitHub Pages).
- **Технологии:** Рекомендуется использовать **SVG Web Animation API** или **Canvas API** для обеспечения высокой производительности (60 FPS).
- **Файл для правки:** `index.html` (весь JS и CSS находится внутри этого файла).

Ниже приведен список смысловых связей, которые нужно визуализировать:

2. `3D Social Platform` <--> `Dialogue Auto-responder` (логика поведения ИИ)
3. `Lazy IDE Coder` <--> `AI Orchestration`

## 6. ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Программист должен предоставить обновленный блок `<script>` и соответствующие стили в `<style>`, которые превратят сухую страницу в живую, реагирующую «матрицу» данных. 
Нужна максимальная оптимизация, чтобы анимации не тормозили на мобильных устройствах.
